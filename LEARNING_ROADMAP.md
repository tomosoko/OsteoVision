# 富士フイルム（トップ医療機器ベンダー）採用への道：学習と実践ロードマップ

このドキュメントは、OsteoVisionプロジェクトを通じて、放射線技師としての高度なドメイン知識と、トップレベルのソフトウェアエンジニアリング（AI・画像処理・システム構築）のスキルを融合させ、**「モダリティベンダーが欲しがる人材」**へ至るための具体的な学習の道標です。

座学ではなく、**「OsteoVisionのコードを書きながら、その裏にある技術的意図を学ぶ（Learning by Doing）」**ことに焦点を当てています。

---

## 📅 ロードマップと現在の進捗

### ✅ Phase 1: 基礎システム構築力 (APIと連携) - 【完了 2026-03-04】

大企業の医療システム（PACS等）に組み込むための疎結合で堅牢なシステムの土台（API）の作り方を学びます。

- [x] **FastAPIの基礎を学ぶ:** FastAPI + uvicornでAPIサーバー構築済み。`/api/health`, `/api/analyze` エンドポイント実装。
- [x] **リクエストとレスポンスの構造:** JSON形式でDICOM画像を受信、角度・QCスコアをJSONレスポンスとして返す。
- [x] **エラーハンドリング (QC Firstの実装):** 回旋角・屈曲角の臨床基準値チェックと警告メッセージ出力を実装済み。

### ✅ Phase 2: AIと幾何学計算の統合 (Computer Vision) - 【完了 2026-03-04】

医療画像ならではの実座標空間（$(x, y)$座標から実際の角度や長さへの変換）とAIの組み合わせを学びます。

- [x] **AIの出力結果（座標）の理解:** YOLOv8-poseが4キーポイント（femur_shaft / medial_condyle / lateral_condyle / tibia_plateau）を出力。mAP50=99.8%達成。
- [x] **Pythonでの幾何学計算 (Trigonometry):** TPA・屈曲角・回旋角を `np.arccos` / `np.arctan2` で計算。`main.py` 実装済み。
- [x] **Config駆動アーキテクチャの構築:** `config.yaml` 中央設定ファイルで閾値・部位設定を管理。ハードコードなし。

### ✅ Phase 3: データエンジニアリング力 (Synthetic Data) - 【完了 2026-03-05】

「データ不足」という最大の壁をエンジニアリングで突破する力を身につけます。

- [x] **3D CTデータと2D DRRの関係性理解:** `OsteoSynth/drr_generator.py` で3Dボリュームレンダリング→DRR投影を実装。`scipy.ndimage.affine_transform` によるリアルタイム回転。
- [x] **微少回旋や位置ズレのシミュレーション:** `yolo_pose_factory.py` で屈曲・回旋・内外反を3軸で独立制御した633枚のDRRを自動生成。スクリューホーム機構（屈曲時脛骨内旋連動、0.12°/deg）も実装済み。

### 🟠 Phase 4: 臨床的妥当性の証明とUI (UX & Validation) - 【進行中 2026-03-05〜】

作ったAIが医療機器（SaMD）として安全であることを証明し、医師が使いやすい形に仕上げます。

- [x] **医療統計（Bland-Altman分析）の実装:** `bland_altman_analysis.py` 実装・デモ実行完了。TPA/屈曲角/回旋角の3グラフ生成済み（`validation_output/`）。実データ収集後にCSVを差し替えるだけで本番解析可能。
- [x] **Next.jsによるフロントエンド構築 (React/TypeScript):** フロントエンド起動・APIとの疎通確認済み（port 3000）。
- [ ] **XAI（Explainable AI）の表示:** Grad-CAM等のヒートマップ表示。ResNet XAIエンジンはロード済み（次フェーズ）。

## 📝 学習の進め方ルール

1.  **「なぜこの技術を使うのか」を必ず問う。**（例: なぜFlaskではなくFastAPIなのか？ なぜ直接計算せずにAPIで分けるのか？）
2.  **コピー＆ペーストは最小限に。** 提供されたコードは必ず手で書き、1行1行の意味を理解する。
3.  **疑問は放置しない。** どんな些細なこと（例: 「この変数名の意味は？」「カッコの種類が違うのはなぜ？」）でもAI（アシスタント）に質問する。
4.  **「技師の視点」を絶えずフィードバックする。** 「現場の感覚としては、こういう時エラーとして弾いてほしい」等、コードに落とし込む仕様を自ら提案する。
